    2021Here we are making vertical timelines in CSS

    Quick overview

    No JS and very primitive use of flex-box containers for page spacing. Some CSS calc() was used, which was previously discouraged ~2019ish but is becoming more accepted as most major browsers and devices use it.

    HTML:

    <div class="timelineContainer"> <div class="lineContainer"> <div class="topCircle"></div> <div class="timeline"></div> <div class="bottomCircle"></div> </div> <div class="textContainer"> <div class="textbox"> <p> <span class="header">13.8 Billion BC</span> Hello Universe! </p> </div> </div> </div>

    As you can see - not very interesting... the whole thing is in its own container, split horizontally via flex by lineContainer and textContainer:

    Layout CSS:

    .timelineContainer {padding:25px 0px;display:flex;flex-wrap:nowrap;background-color:#fff} .lineContainer {width:100px} .textContainer {width:calc(100%-100px)}

    Note that .textContainer uses calc(), which takes 100% usable width, minus the width of the .lineContainer - which is hardcoded rather than rem, em, % or any other variable measurements because it doesn't need to grow or shrink with the page (it would mostly be whitespace anyway).

    Line CSS:

    .topCircle {height:15px;width:15px;background:#111;margin:auto;border-radius:15px 15px 0px 0px} .bottomCircle {height:15px;width:15px;background:#111;margin:auto;border-radius:0px 0px 15px 15px} .timeline {height:calc(100% - 30px);width:5px;background:#111;margin:auto;}

    There's a million ways to do a line with 2 end points, so quickly: Top and bottom ends are just boxes with border-radius applied to top and bottom, the middle part is just a skinny rectangle with a calc() height, 100% minus height of box end-point boxes, all with margin: auto to center.

    Textbox + header CSS:

    .textbox {width:100%} .textbox p {min-height:45px;margin:0px;margin-top:18px;position:relative;left:-35px;font-size:16px;} .textbox p .header {font-weight:bold;font-size:24px;margin-right:10px;}

    Lastly, the actual content! The .textbox class is just an outer wrapper to contain the header (year/date, or whatever you want it to be) and the content. Min-height needs to be set on the paragraph element, or else it will be tiny-height because the position of the p is relative so it can be moved -35px (overlapping the timeline div and going outside of its flexbox). Another note: To avoid the small space after the span just remove the newline.

    That's basically it. Nothing fancy and shouldn't be too hard to copy and use in larger projects, using twig to loop through and output a PHP array fetched from a database should be easy!
