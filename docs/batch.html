    Batch for Windows

    What is batch? What problems does it solve?

    Windows batch files, known synonymously as "batch" or "bat files", are script files that run in the Windows command-prompt (CMD) window, either through the command-line interface, or the explorer Window. Linux users will likely be familiar with the similar bash scripts - one of the most widely used OS level scripting languages, and likely one of the first things that Linux users learn how to use. For Windows users, however, such scripting environments are rarely used.

    Batch scripts allow for quick prototyping of cmd functionality without having to produce an executable. Scripts can do any function that is available within the Windows command-line (which obviously extends to any software that can be downloaded and used within that environment), but for some functionality, and better usability, a GUI interface may be better suited, however in most cases of file or filesystem manipulation where a GUI is not necessary, and where the user needs to apply a function over a wide range of inputs, or produce a large range of outputs, or just produce similar outputs with a specific task, batch scripts can be used.

    Vanilla Batch usage and examples

    Basic Stuff

    Echo output

    Echo allows you to print the succeeding string:

    echo hello world!

    Output:

    hello world!

    You can also echo to a txt file (will be created if it doesn't exist):

    echo hello world!>hello.txt

    Output "hello.txt":

    hello world!

    @ECHO

    The "@" sign forces the succeeding function to not be echo'd, and the "echo off" will disable the echo command by setting the dafault echo behaviour for the entire batch file (so must be set at the top).

    @echo off
echo hello world!

    Output:

    hello world!

    While...

    echo hello world!

    CMD Output:

    C:>echo hello world!
hello world!

    The actual CMD output, without "@echo off", will also render/print the batch functions.

    Drag + Drop files as arguments

    Files can be drag + dropped onto a bat file in the Windows explorer - a faster way to add the file as an argument in the bat script.

    To use the drag + dropped file in the script, you use the same syntax as an argument. "%1" will be the first argument, the number can be changed include any amount of arguments. "%~1" will produce an output without quotes if there is any spaces, useful for when you use the input inside of a quote, e.g. "%~1" can use an input with a space, compared to "%1" which will output ""file"" when a space is present.

    echo %~1

    Output:

    C:\file.txt

    Furthermore, using different notation with this variable will allow you to use more concise arguments in your script, for example "%~n1" will only produce the filename without the location and filename:

    echo %~n1

    Output:

    file
